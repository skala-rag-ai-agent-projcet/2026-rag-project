# 팀 설계 산출물 정리

> 팀원들이 작성한 설계 산출물을 이해하기 쉽게 재정리한 문서

---

## 1. 도메인

**Energy** — Energy Transition / Next-Gen Energy Infrastructure

배터리·에너지저장(ESS)을 시작점으로, AI 기반 에너지 전환 스타트업을 평가한다.

**구체적으로 어떤 스타트업?**
- 배터리 운영 최적화 SW (수명 예측, 충방전 최적화)
- ESS 관리 AI (저장 스케줄링, 경제성 최적화)
- 스마트 그리드 (전력망 예측, 이상 탐지)
- VPP/DR (가상발전소, 수요반응 자동화)
- 에너지 거래 AI (전력 트레이딩)
- 신재생 발전량 예측 (태양광, 풍력)

## 2. 에이전트 구성 (6개)

### 전체 흐름 한눈에 보기

```
스타트업 소싱 → 기술 요약 → 시장/정책 평가 → 경쟁사 비교 → 투자 판단 → 보고서 생성
  (웹서치)      (RAG)       (RAG)         (RAG)       (점수화)      (RAG)
```

### 각 에이전트가 하는 일

#### 🔎 스타트업 소싱 에이전트 — RAG X
> "어떤 스타트업이 있지?"

- 웹서치로 Energy 분야 AI 스타트업 후보를 찾음
- 기업 홈페이지, 뉴스, Crunchbase 등에서 기본 정보 수집
- 이름, 설명, 기술, 펀딩 단계 등을 정리해서 리스트업
- **RAG를 안 쓰는 이유**: 새로운 기업을 발굴하는 단계라 기존 문서가 아니라 웹에서 찾아야 함

#### 🧠 기술 요약 에이전트 — RAG O
> "이 회사 기술이 진짜 괜찮아?"

- 스타트업의 핵심 기술, 차별성, 구현 난이도를 분석
- 에너지 분야 특성상 **수치 스펙**(효율, 에너지밀도, 수명, 비용)을 같이 뽑음
- TRL(기술 성숙도 레벨) 판정
- **RAG 활용**: 기술 논문, 특허 문서, 산업 기술 보고서에서 근거 추출

#### 📈 시장/정책 평가 에이전트 — RAG O
> "시장이 돈이 되나? 정책이 밀어주나?"

- TAM/SAM/SOM 추정, 시장 성장률(CAGR) 분석
- 정책 수혜 여부 (에너지 전환 정책, 보조금, 규제)
- 산업 타이밍 판단 ("지금 들어가도 되나?")
- **RAG 활용**: IEA 리포트, 전력수급기본계획, BNEF 자료 등에서 시장 데이터 추출
- **에너지 산업은 정책 의존도가 높아서** 최신 법령·가이드라인 grounding이 중요

#### ⚔️ 경쟁사 비교 에이전트 — RAG O
> "경쟁자 대비 어디가 나은데?"

- 웹서치로 경쟁사 후보를 넓게 찾음
- 경쟁사 맵, 포지셔닝, 대체기술, 진입장벽 비교
- **RAG 활용**: 이전 에이전트들이 축적한 분석 결과를 다시 참조해서 비교 근거로 활용

#### 🧮 투자 판단 에이전트 — RAG X
> "투자할까 말까? 몇 점이야?"

- 앞의 기술/시장/경쟁 분석 결과를 입력으로 받음
- 9개 항목별 가중 점수를 매김
- 종합 점수 → 투자(Pass) / 보류(Fail) 판정
- **RAG를 안 쓰는 이유**: 핵심이 retrieval이 아니라 스코어링·판단 로직

#### 📝 보고서 생성 에이전트 — RAG O
> "투자위원회에 낼 보고서 만들어줘"

- 모든 분석 결과를 연결해서 보고서 본문 작성
- 문서명·페이지·출처 태깅 유지 → REFERENCE 자동 구성
- **RAG 활용**: 앞선 분석의 근거를 다시 끌어와서 보고서에 인용

## 4. 투자 판단 기준 — 9개 항목 가중 평가

과제의 Scorecard Method + Bessemer Checklist를 참고해서 **9개 항목으로 재편성**.

### 평가표

| # | 항목 | 비중 | 뭘 보는가 |
|---|------|------|---------|
| 1 | **시장 규모·성장성** | 17% | TAM/SAM 크기, 에너지 전환 성장성, 글로벌 확장 가능성 |
| 2 | **SK 사업 시너지** | 15% | SK 계열사와 공동사업 가능성, 밸류체인 편입, 기술 내재화 |
| 3 | **기술 차별성·상용화** | 14% | 성능/효율 우위, PoC/파일럿/상용화 단계, 특허, 모방 난이도 |
| 4 | **창업자/팀** | 12% | 산업 전문성, 실행력, 핵심 인력 구성, 장기 비전 |
| 5 | **문제 해결력/고객 가치** | 10% | 고객 pain point 해결 여부, 기존 방식 대비 개선, 필수성 |
| 6 | **확장성/업사이드** | 10% | 지역/산업 확장 가능성, 플랫폼화, 장기 성장 잠재력 |
| 7 | **고객 지불 의사** | 8% | ROI 명확성, 규제 대응 가치, 유료 PoC/계약 여부 |
| 8 | **수익모델·단위경제성** | 8% | 매출 구조 명확성, 반복매출, 이익률, 회수기간 |
| 9 | **규제·리스크** | 6% | 인허가 필요성, 안전 규제, 공급망 리스크, 법적 분쟁 |
| | **합계** | **100%** | |

### 비중 설계 근거

> "Bessemer는 **무엇을 볼 것인가**를 정하는 데,
> Scorecard는 **무엇에 더 큰 비중을 둘 것인가**를 정하는 데 활용했다."

- **시장(17%)·팀(12%)** 비중 높음 → Scorecard의 시장·팀 중시 경향 반영
- **SK 시너지(15%)** 별도 항목 → 과제 목적상 SK 전략 관점 추가 (팀 독자 항목)
- 나머지 항목 → Bessemer Checklist의 10개 질문을 유사 항목끼리 묶어서 재편성

### 각 항목을 어떤 에이전트가 평가하나

| 항목 | 담당 에이전트 |
|------|-----------|
| SK 사업 시너지 | (별도 판단 로직 또는 투자 판단 에이전트) |
| 시장 규모·성장성 | 시장/정책 평가 |
| 문제 해결력 | 시장/정책 + 스타트업 소싱 |
| 고객 지불 의사 | 시장/정책 |
| 기술 차별성 | 기술 요약 |
| 확장성 | 시장/정책 + 경쟁사 비교 |
| 수익모델 | 스타트업 소싱 + 시장/정책 |
| 규제·리스크 | 시장/정책 + 기술 요약 |
| 창업자/팀 | 스타트업 소싱 |

## 5. 그래프 설계

### 흐름도

```
[START]
   ↓
[스타트업 탐색] → [RAG 기반 정보 검색]
                      ↓
              데이터 충분한가?
              ├─ 부족 → [웹 검색으로 보완] → [분석용 컨텍스트 구성]
              └─ 충분 → [분석용 컨텍스트 구성]
                                ↓
                 ┌──────────────┼──────────────┐
                 ↓              ↓              ↓
          [기술 분석]    [시장/정책 분석]   [경쟁사 분석]     ← 병렬 실행
                 └──────────────┼──────────────┘
                                ↓
                         [투자 판단]
                         점수화 + 판정
                                ↓
                     ┌──── Pass/Fail ────┐
                     ↓                   ↓
              [보고서 생성]          [기각 사유 분석]
                     ↓                   ↓
              [투자 보고서 출력]    [Investment Log 기록]
                     ↓                   ↓
                   [END]              [END]
```

### 과제 가이드와 다른 점

| 항목 | 과제 가이드 | 팀 설계 |
|------|----------|--------|
| 분석 흐름 | 순차 (기술→시장→경쟁사) | **병렬** (기술·시장·경쟁사 동시) |
| 보류 시 | 다음 스타트업으로 **루프 반복** | 기각 사유 기록 후 **종료** |
| RAG 체크 | 없음 | 데이터 충분성 체크 분기 있음 |

> ⚠️ **주의**: 과제 가이드가 명시한 "보류 → 다음 스타트업 반복" 루프가 현재 빠져있음.
> 이건 과제 필수 요구사항이므로 구현 시 추가 필요.

## 6. 보고서 목차 (초안)

```
1. SUMMARY
   - 프로젝트 목적
   - 탐색 대상 도메인
   - 핵심 결과 요약
   - 최종 결론 (투자 후보 발굴 여부, 우선 검토 대상 또는 미선정 사유)

2. 투자 검토 개요
   - 검토 목적 / 범위 (Energy 도메인, 하위 분야)
   - 분석 대상 및 전제
   - 평가 관점 (SK 시너지, 기술성, 시장성, 리스크)

3. 분석 방법 및 평가 기준
   - 웹 검색 + RAG 기반 분석 방식
   - 에이전트 기반 평가 흐름 요약
   - 투자 판단 기준 9개 항목 및 비중

4. 스타트업 탐색 결과
   - 후보 발굴 결과
   - shortlist 선정 과정
   - 최종 검토 대상 기업 개요

5. 기술·시장·정책 분석
   - 핵심 기술 및 상용화 수준
   - 시장 규모 및 성장성
   - 정책 수혜 및 규제 리스크
   - 고객 가치 및 지불 의사

6. 경쟁 구도 및 SK 전략 적합성
   - 경쟁사 비교, 차별화 요소
   - SK 사업과의 시너지

7. 종합 투자 판단
   - 후보별 종합 평가
   - 투자/보류 논리
   - 최종 결론

8. REFERENCE
   - 기관 보고서 / 학술 논문 / 웹페이지
```
